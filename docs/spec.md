## 1. 변곡점 (Pivot Points)의 정의 및 식별 기준

### 1.1. 기본 정의
시장 구조(Market Structure) 분석 및 유동성 구간 파악의 최소 단위로서 고점과 저점의 변곡점을 다음과 같이 정의합니다.
* **스윙 하이 (Swing High, SH):** 특정 캔들의 고점(High)이 주변 기준 캔들들의 고점보다 높은 산봉우리 형태의 고점 구간.
* **스윙 로우 (Swing Low, SL):** 특정 캔들의 저점(Low)이 주변 기준 캔들들의 저점보다 낮은 골짜기 형태의 저점 구간.

### 1.2. 순차적 교차 발생 규칙 (Sequential Alternation)
* **기본 원칙:** 파동의 연속성을 유지하기 위해 확정되는 최종 변곡점은 반드시 **SL → SH → SL → SH** 순으로 상호 교차하며 등장해야 합니다.
* **연속 발생 시 갱신(Update) 처리:**
    * **SH 연속 발생:** SH가 확정된 이후 반대되는 SL이 출현하기 전에 새로운 SH 후보가 다시 나타날 경우, 새로운 고점이 기존 SH의 고점보다 높다면 기존 기록을 취소하고 최신 고점으로 값을 업데이트합니다.
    * **SL 연속 발생:** SL이 확정된 이후 반대되는 SH가 출현하기 전에 새로운 SL 후보가 다시 나타날 경우, 새로운 저점이 기존 SL의 저점보다 낮다면 기존 기록을 취소하고 최신 저점으로 값을 업데이트합니다.

### 1.3. 단일 캔들 내 동시 발생 허용 규칙 (Dual Pivot in Single Candle)
* **기본 원칙:** 극심한 변동성으로 인해 하나의 캔들이 주변 고점과 저점을 동시에 모두 갱신하는 장대봉 또는 아웃사이드 바(Outside Bar)가 발생할 경우, 단일 캔들 내에서 SL과 SH의 동시 발생을 허용합니다.
* **논리적 순서 정렬(Sequencing):** 캔들의 시가(Open)와 종가(Close)의 방향성에 의거하여 내부적인 발생 순서를 강제 부여합니다.
    * **양봉 (Bullish Candle):** **SL → SH** 순서로 정렬
    * **음봉 (Bearish Candle):** **SH → SL** 순서로 정렬

## 2. 4대 스윙 포인트를 이용한 시장 구조 및 추세 판단

가장 최근에 확정된 연속된 4개의 변곡점 (SL 2개, SH 2개)의 상대적 위치를 비교 분석하여 방향성을 정의합니다.

### 2.1. 상승 추세 (Uptrend)
* **판단 조건:** 연속된 2개의 SL 값과 2개의 SH 값이 모두 직전 값보다 상승. (`HL` & `HH`)
* **상태 정의:** 롱(Long) 포지션 진입 우선 고려.

### 2.2. 하락 추세 (Downtrend)
* **판단 조건:** 연속된 2개의 SL 값과 2개의 SH 값이 모두 직전 값보다 하락. (`LL` & `LH`)
* **상태 정의:** 숏(Short) 포지션 진입 우선 고려.

### 2.3. 비추세 및 횡보 (Consolidation / Sideways)
* **판단 조건:** 수렴형(고점 하락 `LH`, 저점 상승 `HL`) 또는 확장형(고점 상승 `HH`, 저점 하락 `LL`).
* **상태 정의:** 횡보 박스(Sideways Box) 상태로 정의, 돌파 또는 박스권 매매 로직 제어 대기.

## 3. 멀티 타임프레임 (MTF) 분석 아키텍처

매매의 신뢰도를 높이고 리스크 대비 보상 비율(RRR)을 극대화하기 위해 두 가지 타임프레임을 조합하여 시장 구조를 매핑합니다. 모든 차트 데이터는 BingX 거래소의 OHLCV 데이터를 기반으로 합니다.

### 3.1. 상위 타임프레임 (HTF: 4시간봉) - 거시적 시장 구조 파악
* **역할:** 큰 흐름에서의 메인 추세(상승, 하락, 비추세)를 결정하고, 주요 유동성 풀 및 오더 블록(Order Block)을 식별합니다.
* **데이터 수집 범위:** BingX API가 제공하는 **최대 길이(Max Lookback Length)**의 OHLCV 데이터를 조회하여 거시적 스윙 포인트를 추적합니다.

### 3.2. 하위 타임프레임 (LTF: 1시간봉) - 미시적 진입 타점 포착
* **역할:** 4시간봉이 결정한 방향성 내에서, 내부 구조(Internal Structure)의 변곡점을 파악하여 최적의 타점을 잡습니다.
* **데이터 수집 범위:** 4시간봉(HTF) 차트에서 식별된 **마지막 스윙 포인트(Swing Point) 구간의 시작 타임스탬프**부터 현재 시점까지의 데이터만 유기적으로 연동하여 1시간봉 변곡점을 다시 그립니다.

## 4. 추세 기반 진입 (EP), 손절 (SL), 익절 (TP) 전략

HTF(4시간봉)에서 판별된 추세 상태에 따라 거래 전략을 차별화합니다. 기본 원칙은 상위 타임프레임의 방향성을 하위 타임프레임에서 추종하는 것입니다.

### 4.1. 상승 추세 (Uptrend) 거래 전략 (Long Position)
4시간봉 기준 `HL`과 `HH`가 확인된 상태에서, 1시간봉의 눌림목(Pullback)을 활용해 매수(Long) 진입합니다.
* **포지션 진입 (EP):** 1시간봉에서 새로운 스윙 로우(SL)를 형성하고 반등을 시작하는 시점. 해당 타점이 4시간봉의 주요 강세 오더 블록(OB)이나 공정 가치 갭(FVG) 구간과 겹칠 때 신뢰도가 극대화됩니다.
* **손절 (SL):** 진입 근거가 된 최근 1시간봉 스윙 로우(SL) 직하단, 또는 구조적 훼손을 의미하는 4시간봉 스윙 로우 이탈 지점.
* **익절 (TP):** 직전 4시간봉 스윙 하이(SH) 구간 (내부 유동성 확보). 돌파 기대 시 분할 익절 후 트레일링 스탑 적용.

### 4.2. 하락 추세 (Downtrend) 거래 전략 (Short Position)
4시간봉 기준 `LL`과 `LH`가 확인된 상태에서, 1시간봉의 되돌림(Retracement)을 활용해 매도(Short) 진입합니다.
* **포지션 진입 (EP):** 1시간봉에서 새로운 스윙 하이(SH)를 형성하고 하락으로 꺾이는 시점. 해당 타점이 4시간봉의 약세 오더 블록(OB)이나 FVG 구간 저항과 일치할 때 진입합니다.
* **손절 (SL):** 진입 근거가 된 최근 1시간봉 스윙 하이(SH) 직상단, 또는 4시간봉 스윙 하이 돌파 지점.
* **익절 (TP):** 직전 4시간봉 스윙 로우(SL) 구간.

### 4.3. 비추세 및 횡보 (Consolidation / Sideways Box) 거래 전략
4시간봉(HTF)의 스윙 포인트들이 수렴하거나 확장하며 일정한 방향성을 잃은 구간입니다. 추세가 없으므로 기존의 추세 추종 로직은 일시 정지되며, 가격의 평균 회귀(Mean Reversion) 성질을 이용한 RSI 기반 박스권 매매 로직이 활성화됩니다.

* **핵심 필터 (Condition):** 4시간봉 횡보 상태 확인 시, 1시간봉(LTF) 진입 로직에 **RSI (기간: 14) 필터**를 강제 적용.
* **포지션 진입 (EP):** 1. **박스권 하단 매수 (Long):** 가격이 4시간봉 기준 박스권 하단(SL 지지대) 부근에 도달하고, 1시간봉 RSI가 과매도 구간 (30 이하)에 진입한 후 다시 30을 돌파하며 상승할 때 진입.
    2. **박스권 상단 매도 (Short):** 가격이 4시간봉 기준 박스권 상단(SH 저항대) 부근에 도달하고, 1시간봉 RSI가 과매수 구간 (70 이상)에 진입한 후 다시 70을 이탈하며 하락할 때 진입.
* **손절 (SL):** 휩소(Whipsaw) 꼬리 이탈을 방지하기 위해, 단순 가격 이탈이 아닌 1시간봉 캔들이 박스권 경계선 밖에서 실체(Body)로 마감(Close)되었을 때 손절 처리.
* **익절 (TP):** * **1차 익절 (TP1):** 박스권의 중심축 (Mid-line). 여기서 포지션의 50%를 청산하고 본절가(Break-even)로 SL 이동.
    * **2차 익절 (TP2):** 박스권의 반대편 경계선 도달 시점, 또는 RSI가 반대편 극한값(롱일 경우 과매수, 숏일 경우 과매도)에 도달했을 때 전량 청산.

## 5. 리스크 관리 및 포지션 사이징 (Risk Management & Position Sizing)

안정적인 누적 수익을 달성하고 연패(Drawdown) 시 계좌의 파산을 방지하기 위해, 1회 거래에서 감수하는 최대 손실액을 전체 자본의 고정된 비율로 제한하는 **고정 리스크 모델(Fixed Fractional Risk Model)**을 적용합니다.

### 5.1. 기본 설정 (Base Parameters)
* **1회 최대 손실 허용폭 (Risk Per Trade):** 총 자본(Total Equity)의 **1%**
* **사용 레버리지 (Leverage):** **10배 (10x)** 고정

### 5.2. 포지션 규모 역추산 로직 (Position Size Reverse Calculation)
차트 구조(MTF 분석)에 의해 진입가(EP)와 손절가(SL)가 확정되면, 해당 구간에서 손절이 발생했을 때 정확히 총 자본의 1%만 잃도록 투입 금액(Position Size)을 역으로 계산합니다.

**계산 공식:**
1. **최대 허용 손실금 (Risk Amount):** `총 자본 * 0.01`
2. **손절 거리 (SL Distance %):** `|EP - SL| / EP`
3. **총 포지션 규모 (Total Position Value):** `최대 허용 손실금 / 손절 거리(%)`
4. **실제 필요 증거금 (Required Margin):** `총 포지션 규모 / 레버리지(10)`
5. **실제 주문 수량 (Coin Quantity):** `총 포지션 규모 / EP`

### 5.3. 안전 장치 및 예외 처리 (Safety Checks)
* **강제 청산(Liquidation) 방어 로직:** 10배 레버리지 사용 시 진입가 대비 약 10% 역행하면 강제 청산이 발생합니다. 만약 1시간봉 차트의 변동성이 너무 커서 계산된 손절 거리(SL Distance)가 9%를 초과할 경우, 손절가(SL)에 도달하기 전에 거래소가 강제 청산을 집행할 위험이 있습니다.
* **조치 사항:** 이 조건이 발생하면 시스템은 해당 거래를 위험 타점으로 간주하고 진입을 취소(Skip)하거나, 예외적으로 레버리지를 낮춰 청산가를 SL보다 멀리 이격시키는 안전 루틴을 실행합니다.

## 7. 시스템 아키텍처 및 구현 기술 명세 (Technical Specifications)

### 7.1. 개발 스택 및 데이터 소스
* **매매를 진행하는 Bot 과 Back testing결과를 확인하기위한 ui로 구성**
* **Bot 개발 언어:** **Python** 기반 엔진 구축 (BingX API가 Python 환경과 안정적으로 연동됨에 기반함)
* **Back testing 개발 언어** : **Backend - python fast api**, **Frontend - React** 기반
* **거래소 연동:** 데이터 수집(OHLCV) 및 주문 체결(EP, SL, TP) 프로세스는 모두 **BingX API** 패키지를 경유하여 처리.

### 7.2. 예기치 못한 종료 대응 및 상태 보존 (State Persistence & Crash Recovery)
서버 다운, 네트워크 유실 등 예기치 않게 트레이딩 봇 시스템이 오프(Off)되었을 때 데이터 연속성이 깨지는 문제를 방지하기 위해 **상태 파일(State File)** 아키텍처를 도입합니다.
* **지속성 데이터 범위:** 1. 확정 및 미확정된 실시간 스윙 포인트(HTF/LTF 변곡점의 인덱스, 가격, 타입 정보) 데이터
    2. 현재 실행 중인 포지션 현황(진입가, 타깃 손절가/익절가, 계약 수량 등)
* **복구 메커니즘:** 시스템은 매 캔들 마감 및 주문 상태 변경 시마다 상태 파일을 내부 스토리지에 실시간 업데이트(Write)합니다. 시스템 인스턴스가 비정상 종료된 후 재시작(Reboot)될 때, 초기화 단계에서 이 상태 파일을 로드(Read)하여 이전의 마켓 구조 상태와 포지션 트래킹 모듈을 완벽하게 동기화 및 복구합니다.
